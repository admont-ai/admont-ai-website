Security

Share knowledge broadly without losing control of who can see what.

Admont AI is designed for teams that want documentation to be easy to create and easy to reuse, while still respecting access boundaries, repository visibility, and assistant integrations. Security needs to be understandable in practice, not just described in marketing terms.

  • Authentication and session protection
  • Repository and file-level visibility controls
  • MCP access with dedicated authorization flow
  • Clear boundaries for shared knowledge

Key security considerations

  • Protect against unauthorized access to repositories, documents, and artefacts.
  • Control what authorized users can access through permissions on documents and other artefacts.
  • Prevent information leakage by applying access control consistently across retrieval and search.
  • RAG search only includes knowledge the current user is actually allowed to access.

Principles

Security is part of the workflow, not an afterthought.

Least Exposure

Keep access scoped so teams and assistants only see the repositories and files they should be able to use.

Auditability

Use readable repositories, clear history, and inspectable code paths instead of opaque knowledge storage.

Operational Trust

Make the security model understandable enough that technical teams can review and adapt it themselves.

Controls

Key controls around access, repositories, and assistant integrations.

OAuth-based sign-in

Browser authentication uses a modern authorization flow so teams can establish identity before exposing protected content.

Protected sessions

Session handling is designed to support authenticated use without treating the app like a public anonymous content surface.

Repository visibility boundaries

Separate public, authenticated, and administrative scopes so access can reflect how knowledge should actually be shared.

File and path permissions

Apply path-aware controls to keep sensitive areas protected even when broader repositories contain useful shared documentation.

MCP authorization flow

External assistant access is handled through a dedicated flow so machine clients are not treated like an afterthought.

Reviewable behavior

Teams need a security model they can understand, validate, and operate with confidence instead of relying on vague assurances.

Admont AI helps teams share knowledge with clearer access, visibility, and assistant controls.
