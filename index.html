Product

A new documentation workflow, not just a new editor.

The core idea behind Admont AI is recursive documentation generation: AI helps create documentation, that documentation is stored as structured knowledge, and the accumulated knowledge base becomes the source for better future documentation. Git and Markdown are not the product story on their own. They are the best substrate for keeping that loop transparent, durable, and easy to evolve.

Features

Everything is built around the RAG authoring loop.

AI-assisted generation

Generate new documentation from prompts, existing documents, and retrieved repository context instead of writing every page from scratch.

Compounding knowledge base

Each accepted document becomes material for later retrieval, so the system improves as the documentation corpus grows.

Hybrid retrieval

Combine full-text and semantic search to pull the most relevant context into answers, generation, and editing flows.

Markdown as working memory

Keep documentation in plain Markdown so generated knowledge stays durable, inspectable, and easy to reuse outside the app.

Team review and history

Review, diff, publish, and recover generated content with a workflow that treats documentation as a shared organizational asset.

MCP and assistant access

Expose the same retrieval-ready documentation base to external AI assistants through the Model Context Protocol.

Security

Built for teams that need controlled knowledge sharing.

Admont AI is designed for teams and organizations where documentation should be easy to generate and reuse, but never lose on access control. Security runs through the full workflow: authentication, repository visibility, file-level permissions, and MCP access for external assistants.

OAuth with protected sessions

Browser sign-in uses OAuth 2.0 with PKCE, short-lived auth state, and signed JWT sessions for authenticated access.

Repository and role controls

Separate public, authenticated, and admin-only operations so teams can expose the right knowledge to the right people.

File-level permissions

Apply path-based permissions for files and folders with owner, user, group, and inherited defaults.

Secure MCP access

External assistants connect through a dedicated MCP OAuth flow with dynamic client registration and PKCE-protected token exchange.

Read-only and dotfile protection

Lock repositories down for safe consumption and block access to sensitive internal paths like Git metadata and permission files.

Low-leakage responses

Permission denials can resolve as not found, helping reduce path enumeration and accidental disclosure.

Homage

Named in tribute to the Admont Abbey library.

Admont AI takes its name from the Admont Abbey library in Austria, a place shaped by the long human tradition of collecting, preserving, and sharing knowledge. The project is a modern homage to that idea: documentation as a living body of knowledge that can be expanded, retrieved, and passed forward.
