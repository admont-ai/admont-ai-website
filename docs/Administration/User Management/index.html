User Management

Admont-AI supports two types of user accounts: internal users (managed by the platform) and external users (linked to OAuth providers). Both types can be assigned roles and managed through the admin panel.

Roles

Role Capabilities
super_admin All roles; bypasses all permission checks
system_admin Manage auth providers, LLM providers, search providers
user_admin Manage internal/external users and groups
repo_admin Manage repositories (add, configure, re-clone, reindex)

Super admins automatically have all roles. Roles can be assigned to individual users or to groups (all group members inherit the group's roles).

Internal Users

Internal users authenticate with email and password, managed directly by Admont-AI via Ory Hydra.

Creating an Internal User

  1. Open the Admin panel from the header.
  2. Go to the Internal Users tab.
  3. Click Add User.
  4. Enter email, first name, last name, and a password.
  5. Select roles and optionally enable super admin.

Internal users support:

  • Password management — Admins can force password expiry, requiring the user to set a new password on next login.
  • Two-factor authentication (TOTP) — Users can enable TOTP-based 2FA. Admins can reset a user's TOTP configuration if they lose access to their authenticator.
  • Account suspension — Suspended users cannot log in.

Editing and Deleting

Click a user in the list to edit their details, roles, or suspension status. Use the delete button to remove a user permanently.

External Users

External users authenticate via a configured OAuth provider (Google, GitHub, Okta, Azure AD, etc.). Their accounts are created in Admont-AI and linked to the provider.

Creating an External User

  1. Go to the External Users tab in the admin panel.
  2. Click Add User.
  3. Select the OAuth provider from the dropdown.
  4. Enter the user's email, first name, and last name.
  5. Assign roles.

External users inherit their identity from the OAuth provider. They cannot change their password or enable TOTP through Admont-AI.

Groups

Groups let you manage roles and permissions for multiple users at once.

Creating a Group

  1. Go to the Groups tab in the admin panel.
  2. Click Add Group.
  3. Enter a name and optional description.
  4. Add members by searching for users (both internal and external).
  5. Assign roles to the group.

All members of a group inherit the group's roles. A user who belongs to multiple groups receives the union of all group roles.

Group Uses

Groups serve two purposes:

  1. Role assignment — Admin roles (system_admin, user_admin, repo_admin) assigned to the group apply to all members.
  2. File permissions — Groups can be referenced in .file-permissions.yaml to grant file-level access to all members.

API Endpoints

Method Path Description
GET /admin/users List all users (internal + external)
GET /admin/users/internal List internal users
POST /admin/users/internal Create internal user
PUT /admin/users/internal/:email Update internal user
DELETE /admin/users/internal/:email Delete internal user
DELETE /admin/users/internal/:email/totp Reset TOTP
GET /admin/users/external List external users
POST /admin/users/external Create external user
PUT /admin/users/external/:provider/:email Update external user
DELETE /admin/users/external/:provider/:email Delete external user
GET /admin/groups List groups
POST /admin/groups Create group
PUT /admin/groups/:name Update group
DELETE /admin/groups/:name Delete group

All user management endpoints require the user_admin role.
