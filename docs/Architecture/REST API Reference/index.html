REST API Reference

The REST API is organized around repositories, files, folders, drafts, search, and administration. Fuego generates a full OpenAPI 3.1 specification available at /swagger/openapi.json.

Core Endpoints

Method Path Description
Repositories
GET /repos List all accessible repositories
GET /repos/:repo List files and folders in a repository
Files
GET /repos/:repo/file/*path Read file content (returns draft if one exists)
POST /repos/:repo/file/*path Create a new file
PUT /repos/:repo/file/*path Update file content
DELETE /repos/:repo/file/*path Delete a file
PATCH /repos/:repo/file/*path Move a file to a new location
PUT /repos/:repo/rename/*path Rename a file
POST /repos/:repo/upload/*path Upload a binary file (base64 encoded)
File Info & History
GET /repos/:repo/fileinfo/*path Get file metadata (author, date, size, permissions)
GET /repos/:repo/filehistory/*path Get commit history for a file
GET /repos/:repo/filediff/*path Get diff between a commit and HEAD
GET /repos/:repo/fileatcommit/*path Get file content at a specific commit
Drafts
PUT /repos/:repo/draft/*path Save or update a draft
POST /repos/:repo/draft/publish/*path Publish a draft (three-way merge)
DELETE /repos/:repo/draft/*path Discard a draft
Folders
POST /repos/:repo/folder/*path Create a folder
PUT /repos/:repo/folder/*path Rename a folder
DELETE /repos/:repo/folder/*path Delete a folder
PATCH /repos/:repo/folder/*path Move a folder
Search
POST /repos/search Search documents (fulltext, semantic, or hybrid)
GET /repos/search/status Get search index status per repository
Permissions
GET/PUT/DELETE /repos/:repo/permissions/*path Manage file-level permissions
AI
GET /ai/models List available AI models
POST /ai Execute AI operation (ask, generate, polish)
Auth
GET /auth/login Initiate Google OAuth flow
GET /auth/callback Handle OAuth callback
Admin
POST /admin/repos Add a new repository
PUT /admin/repos/:slug/settings Update repository settings
POST /admin/repos/:slug/reindex Rebuild search index

Middleware Stack

All requests pass through CORS handling and gzip compression (except MCP SSE endpoints, which require uncompressed streaming). Request logging uses Logrus with structured fields.
