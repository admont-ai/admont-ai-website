Frontend

The frontend is a React 19 single-page application built with TypeScript, Vite, and Tailwind CSS. It uses shadcn/ui (built on Radix UI primitives) for base components and MDXEditor (Lexical-based) for WYSIWYG markdown editing.

State Management

The app uses React Context and custom hooks rather than an external state management library:

  • AuthContext — Global authentication state: user identity, permissions, JWT lifecycle, auto-logout before token expiry.
  • AiLogContext — AI operation history and model selection.
  • Custom hooks — useRepos, useDocuments, useDocumentContent, useDocumentSave, useDocumentMutations, useFolders, useFileHistory, useFilePermission encapsulate API calls and loading/error states.

API Communication

All API calls go through a centralized authFetch() wrapper that injects the JWT Bearer token from localStorage, handles 401 responses by dispatching an auth:unauthorized event (triggering logout), and surfaces errors as toast notifications.

In development, Vite proxies API paths (/repos, /auth, /me, /admin, /ai) to the backend at localhost:8080.

Routing

The app uses URL-based SPA routing without a router library. The URL pattern /{repo}/{filePath} is parsed on load and updated via window.history.replaceState() on navigation. Deep linking is fully supported. File type is detected by extension and routes to the appropriate editor component.

Key UI Features

  • Sidebar file tree with drag-and-drop for moving files and folders
  • Hybrid search bar with configurable modes (fulltext, semantic, hybrid) and 300ms debounce
  • Draft workflow with auto-save, publish, and discard actions
  • File history panel showing commit log with diff view and restore capability
  • AI assistant supporting ask, generate, and polish modes with model selection and token usage tracking
  • Permission management dialogs for file and folder access control
