Configuration

Admont-AI is configured via a YAML file (default: config.yaml) loaded by the Viper configuration library. The configuration covers the server, authentication, repositories, LLM providers, search engines, and CORS.

Server

server:
  port: 8080
  host: "0.0.0.0"

Database

database:
  url: "postgres://user:password@localhost:5432/admont?sslmode=disable"

The database URL is a standard PostgreSQL connection string. The server runs migrations automatically on startup.

Authentication

auth:
  jwt_secret: "your-secret-key"
  providers:
    - name: google
      client_id: "..."
      client_secret: "..."

Admont-AI supports multiple OAuth 2.0 providers. Each provider needs a name, client_id, and client_secret. Supported providers include: Google, GitHub, GitLab, Apple, Auth0, Azure AD, Bitbucket, Cognito, Discord, Facebook, LinkedIn, Microsoft, Okta, OpenID Connect, and Slack.

The jwt_secret is used to sign JWT tokens (HMAC-SHA256). It should be a strong random string.

Repositories

repos:
  - slug: "my-docs"
    name: "My Documentation"
    url: "https://github.com/org/docs.git"
    branch: "main"
    description: "Team documentation"
    public_access: false
    search_provider: "pgvector"
Field Description
slug URL-safe identifier for the repo
name Display name
url Git clone URL (supports HTTPS with embedded credentials, SSH)
branch Branch to track (default: main)
description Short description shown in the UI
public_access If true, anonymous users can read content
search_provider Search engine to use: pgvector, elasticsearch, or typesense

LLM Providers

llm:
  providers:
    - name: anthropic
      api_key: "sk-ant-..."
      models:
        - id: "claude-sonnet-4-20250514"
          name: "Claude Sonnet"
    - name: openai
      api_key: "sk-..."
      models:
        - id: "gpt-4o"
          name: "GPT-4o"
    - name: ollama
      base_url: "http://localhost:11434"
      models:
        - id: "llama3"
          name: "Llama 3"

Supported LLM providers: Anthropic, OpenAI, Google, DeepSeek, Meta, Mistral, Ollama (self-hosted), Perplexity, and xAI. Each provider requires at minimum a name and either an api_key or base_url.

LLM configuration can be updated at runtime via the admin panel without restarting the server.

Search Engines

pgvector (Default)

search:
  pgvector:
    enabled: true

Uses PostgreSQL with the pgvector extension for both full-text and semantic search. Embeddings are generated locally using the all-MiniLM-L6-v2 ONNX model.

Elasticsearch

search:
  elasticsearch:
    url: "http://localhost:9200"

Typesense

search:
  typesense:
    url: "http://localhost:8108"
    api_key: "..."

CORS

cors:
  allowed_origins:
    - "http://localhost:5174"
    - "https://your-domain.com"

CORS origins must be explicitly configured. The server will not start if no origins are specified.

Users and Roles

users:
  super_admins:
    - "admin@example.com"
  admins:
    - "editor@example.com"
  • super_admin — Bypasses all permission checks, full access to everything.
  • admin — Access to the admin panel for managing repos, users, groups, and LLM settings.
