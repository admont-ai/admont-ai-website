Installation

Prerequisites

Component Required Version Purpose
Go 1.25+ Backend server
Node.js 22+ Frontend build and dev server
PostgreSQL 17+ Database for users, search index, and conversations
pgvector 0.7+ PostgreSQL extension for semantic search vectors
Git 2.x Repository management

Backend Setup

1. Clone the repository

git clone <repo-url> admont-ai-api
cd admont-ai-api

2. Install Go dependencies

go mod download

3. Set up the database

Create a PostgreSQL database and enable the pgvector extension:

CREATE DATABASE admont;
\c admont
CREATE EXTENSION IF NOT EXISTS vector;

4. Run database migrations

Migrations are in internal/store/migrations/. Apply them in order:

psql -d admont -f internal/store/migrations/000001_create_users.up.sql
psql -d admont -f internal/store/migrations/000002_search_chunks.up.sql
psql -d admont -f internal/store/migrations/000003_ai_conversations.up.sql

5. Create a configuration file

Copy the example configuration and edit it:

cp config.example.yaml config.yaml

See Configuration for details on all available settings.

6. Build and run

go build -o admont-ai .
./admont-ai

Or run directly:

go run .

The server starts on port 8080 by default.

Frontend Setup

1. Navigate to the UI directory

cd admont-ai-ui

2. Install dependencies

npm install

3. Start the development server

npm run dev

The frontend starts on port 5174 and proxies API requests to the backend at localhost:8080.

4. Build for production

npm run build

The built files are output to dist/ and can be served by any static file server or by the Go backend itself.

ONNX Runtime (Optional)

For local semantic search embeddings (pgvector backend), the server uses ONNX Runtime with the all-MiniLM-L6-v2 model. The model files are downloaded automatically on first startup if the pgvector search provider is configured.
