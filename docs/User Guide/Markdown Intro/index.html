Markdown Intro

Overview

Markdown is a lightweight markup language created by John Gruber in 2004. It allows you to write formatted text using a plain-text editor with a simple, readable syntax that can be converted to HTML and many other formats. Markdown has become the de facto standard for writing documentation, README files, forum posts, and content across platforms like GitHub, GitLab, Reddit, and Stack Overflow.

The core philosophy behind Markdown is that the source text should be readable as-is, without needing to be rendered. Unlike HTML or LaTeX, Markdown files look clean and understandable even in their raw form.

Advantages

  • Easy to learn — The syntax is minimal and intuitive. Most people can pick it up in minutes.
  • Readable as plain text — Markdown files are human-readable even without rendering, making them ideal for version control.
  • Portable — Markdown is just plain text, so it works on any operating system and with any text editor.
  • Widely supported — GitHub, GitLab, Notion, Jira, Slack, and countless other tools support Markdown natively.
  • Convertible — Tools like Pandoc can convert Markdown to HTML, PDF, DOCX, LaTeX, and many other formats.
  • Version control friendly — Since Markdown is plain text, diffs are clean and meaningful in Git.
  • Focused writing — The simple syntax keeps you focused on content rather than formatting.

Limitations

  • Limited styling — Markdown supports basic formatting but lacks fine-grained control over layout, colors, or typography.
  • No standard specification — Multiple flavors exist (CommonMark, GitHub Flavored Markdown, MultiMarkdown), which can lead to inconsistencies across platforms.
  • Complex tables are difficult — While basic tables are supported, complex layouts with merged cells or nested content are hard or impossible.
  • No native support for advanced features — Things like footnotes, math equations, or diagrams require extensions or specific flavors.
  • Embedded HTML as a workaround — When Markdown falls short, you often need to fall back to raw HTML, which defeats the purpose of simplicity.

Tutorial: Writing Markdown Files

Headings

Use # symbols to create headings. The number of # symbols determines the heading level.

# Heading 1
## Heading 2
### Heading 3
#### Heading 4
##### Heading 5
###### Heading 6

Text Formatting

**bold text**
*italic text*
***bold and italic***
~~strikethrough~~
`inline code`

Links

[Link text](https://example.com)
[Link with title](https://example.com "Title text")

Lists

Unordered lists use -, *, or +:

- Item one
- Item two
  - Nested item
  - Another nested item
- Item three

Ordered lists use numbers:

1. First item
2. Second item
3. Third item

Blockquotes

> This is a blockquote.
>
> It can span multiple lines.

Code Blocks

Use triple backticks for fenced code blocks. Optionally specify the language for syntax highlighting:

```python
def hello():
    print("Hello, world!")
```

Tables

| Column 1 | Column 2 | Column 3 |
|----------|----------|----------|
| Row 1    | Data     | Data     |
| Row 2    | Data     | Data     |

You can align columns using colons:

| Left     | Center   | Right    |
|:---------|:--------:|---------:|
| aligned  | aligned  | aligned  |

Horizontal Rules

Use three or more dashes, asterisks, or underscores:

---
***
___

Task Lists

- [x] Completed task
- [ ] Incomplete task
- [ ] Another task

Embedding External Media

Images

Images use the same syntax as links, prefixed with !:

![Alt text](path/to/image.png)
![Alt text](path/to/image.jpg "Optional title")

You can reference images from a URL:

![Logo](https://example.com/logo.png)

Or use a relative path for images stored alongside your Markdown files:

![Diagram](./images/diagram.png)

To control image size, you need to fall back to HTML:

<img src="image.png" alt="Alt text" width="400">

Videos

Markdown has no native video embedding syntax. There are several workarounds:

Link to the video with a thumbnail image:

[![Video title](https://img.youtube.com/vi/VIDEO_ID/0.jpg)](https://www.youtube.com/watch?v=VIDEO_ID)

This displays a clickable thumbnail that opens the video.

Embed using HTML (where supported):

<video width="640" controls>
  <source src="video.mp4" type="video/mp4">
</video>

YouTube/Vimeo iframe (where HTML is allowed):

<iframe width="560" height="315" src="https://www.youtube.com/embed/VIDEO_ID" frameborder="0" allowfullscreen></iframe>

Note: Not all Markdown renderers allow raw HTML. GitHub, for example, strips <iframe> and <video> tags for security reasons. In those cases, linking to the video is the most reliable approach.

Audio

Similar to video, audio requires HTML:

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
</audio>

SVG and Other Formats

SVGs can be embedded as images:

![Diagram](diagram.svg)

Or inline using HTML for more control:

<img src="diagram.svg" alt="Diagram" width="600">
