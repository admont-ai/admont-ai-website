File History

Every document in Admont-AI is version-controlled with Git. You can view the full commit history of any file, compare changes between versions, and restore previous versions.

Viewing History

  1. Open a document.
  2. Click the menu (three dots) in the editor toolbar.
  3. Select Version history.

The history panel opens on the right, showing a chronological list of commits for the file. Each entry displays:

  • Commit message — What was changed
  • Author — Who made the change
  • Date — When the change was committed
  • Commit hash — The abbreviated Git commit identifier

Comparing Versions

Click any commit in the history list to see a diff view. The editor switches to a side-by-side comparison showing:

  • The selected commit's version on the left
  • The current (HEAD) version on the right
  • Additions highlighted in green
  • Deletions highlighted in red

The diff header shows the commit details: message, author, and hash.

Restoring a Version

To restore a previous version:

  1. Select the commit you want to restore in the history panel.
  2. Click the Restore button in the diff view header.

This copies the content from that commit into a new draft. You can review the restored content, make further edits if needed, and publish it as a new commit. The original history is preserved — restoring does not rewrite Git history.

Closing the History View

Click Close in the diff view header to return to the normal editor view.

Technical Details

  • History is fetched via GET /repos/:repo/filehistory/*path, which returns the Git log for that file.
  • Diffs are generated via GET /repos/:repo/filediff/*path?commit=<hash>, comparing the specified commit to HEAD.
  • File content at a specific commit is retrieved via GET /repos/:repo/fileatcommit/*path?commit=<hash>.
  • Restoring a version creates a draft with the old content — it does not force-push or rebase.
