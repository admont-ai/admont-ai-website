Development Setup

This guide covers the local development workflow for contributing to Admont-AI.

Prerequisites

  • Go 1.25+ — Backend
  • Node.js 22+ — Frontend
  • PostgreSQL 17+ with pgvector extension — Database
  • Docker (optional) — For running PostgreSQL, Ory Hydra, and LanguageTool

Quick Setup with Docker Compose

The easiest way to run the supporting services:

cd admont-ai-api/docker-compose
docker-compose up -d

This starts:

  • PostgreSQL (port 5433) — with pgvector extension, user admin, database admont-ai
  • Ory Hydra (ports 4444/4445) — OAuth 2.0 / OIDC server for internal authentication
  • LanguageTool (port 8081) — Grammar and spelling checker (optional)

Running the API

cd admont-ai-api

# Install dependencies
go mod download

# Create .env with required secrets
echo 'JWT_SECRET=your-hex-encoded-secret' > .env
echo 'AUTH_BASE_URL=http://localhost:8080' >> .env

# Run the server
make run
# or: go run .

The API starts on http://localhost:8080. On first run it:

  1. Connects to PostgreSQL and runs migrations automatically
  2. Generates a JWT secret if not provided
  3. Initializes authentication providers
  4. Clones configured repositories in the background
  5. Starts the search indexer

Running the Frontend

cd admont-ai-ui

# Install dependencies
npm install

# Start dev server
npm run dev

The frontend starts on http://localhost:5174 with Vite's hot module replacement. API requests are proxied to localhost:8080 via the Vite dev server config.

Proxied Paths

The Vite proxy forwards these paths to the backend:

  • /auth/* — Authentication
  • /me — User info
  • /admin — Admin API
  • /llm — LLM operations
  • /checker — Grammar checker
  • /conversations — AI conversations
  • /repos — Repository and file operations

Database Migrations

Migrations are in internal/store/migrations/ and run automatically on server startup. To apply manually:

psql -h localhost -p 5433 -U admin -d admont-ai \
  -f internal/store/migrations/000001_initial_schema.up.sql

ONNX Model Setup (Search)

For local semantic search with pgvector, you need the all-MiniLM-L6-v2 ONNX model:

python3 -m venv .venv
source .venv/bin/activate
pip install "optimum[onnxruntime]" onnx onnxruntime transformers torch
python3 export_model.py

curl -fsSL -o ./models/vocab.txt \
  https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2/resolve/main/vocab.txt

Configure the model paths in .env:

SEARCH_ONNX_RUNTIME_PATH=/opt/homebrew/lib/libonnxruntime.dylib
SEARCH_MODEL_PATH=models/model.onnx
SEARCH_VOCAB_PATH=models/vocab.txt

Environment Variables

Variable Description
JWT_SECRET Hex-encoded secret for signing JWTs
AUTH_BASE_URL Base URL for OAuth callbacks (e.g., http://localhost:8080)
LANGUAGETOOL_URL Grammar checker URL (optional)
SEARCH_ONNX_RUNTIME_PATH Path to libonnxruntime shared library
SEARCH_MODEL_PATH Path to ONNX model file
SEARCH_VOCAB_PATH Path to tokenizer vocab file
ADMONT_ENCRYPTION_KEY Hex-encoded 32-byte key for encrypting secrets in DB

Building

# Backend
cd admont-ai-api
go build -o admont-ai .

# Frontend
cd admont-ai-ui
npm run build   # outputs to dist/

Running Tests

cd admont-ai-api
go test ./...

Tests exist for permissions resolver, chunker, embedder tokenizer, and Confluence importer.

Project Structure

API (admont-ai-api)

Directory Purpose
config/ YAML config loading via Viper
internal/auth/ OAuth providers and JWT
internal/draft/ Draft storage and three-way merge
internal/llm/ LLM provider integrations (9 providers)
internal/mcp/ MCP server with SSE transport
internal/permissions/ File-level access control
internal/pg_vector/ Search: chunker, embedder, indexer
internal/repo/ Repository backends (git, S3)
internal/request_handler/ HTTP endpoint handlers
internal/store/ PostgreSQL data access layer
main.go Entry point and route registration

UI (admont-ai-ui)

Directory Purpose
src/components/layout/ Page-level components (editors, sidebar, dialogs)
src/components/ui/ shadcn/ui base components
src/contexts/ React context providers (auth)
src/hooks/ Custom React hooks (API calls, state)
src/lib/ Utilities (auth-fetch, PDF export, Monaco config)
src/plugins/ MDXEditor plugins (math, video)
