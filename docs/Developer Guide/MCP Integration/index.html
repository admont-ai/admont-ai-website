MCP Integration

The Model Context Protocol (MCP) allows AI assistants like Claude to interact with your wiki programmatically. The MCP server exposes 22 tools for reading, writing, and searching documents.

Connecting Claude Code

Automatic OAuth (Recommended)

claude mcp add --transport sse md-wiki http://localhost:8080/mcp/sse

Or scope it to your project:

claude mcp add -s project --transport sse md-wiki http://localhost:8080/mcp/sse

On first connection, Claude Code will open a browser window for OAuth authentication. After authenticating, the MCP session is established automatically.

Manual Configuration

Add to ~/.claude/settings.json:

{
  "mcpServers": {
    "md-wiki": {
      "type": "sse",
      "url": "http://localhost:8080/mcp/sse"
    }
  }
}

With Pre-existing Token

If you already have a JWT token:

claude mcp add --transport sse md-wiki http://localhost:8080/mcp/sse \
  --header "Authorization: Bearer YOUR_JWT_TOKEN"

Authentication Flow

The MCP server uses its own OAuth flow with dynamic client registration (RFC 7591) and PKCE:

  1. Client connects to GET /mcp/sse without a token.
  2. Server responds with 401 and a WWW-Authenticate header pointing to discovery endpoints.
  3. Client discovers OAuth metadata via /.well-known/oauth-authorization-server.
  4. Client registers dynamically at POST /mcp/register to get a client_id.
  5. Client redirects the user to GET /mcp/authorize with a PKCE challenge.
  6. User authenticates via OAuth provider (opens in browser).
  7. Server issues an authorization code (5-minute TTL).
  8. Client exchanges the code at POST /mcp/token for a JWT.
  9. Client reconnects to SSE with the JWT in the Authorization header.

Available Tools

Repository Tools

Tool Description
list_repos List all accessible repositories
get_repo List all files and folders in a repository

File Read Tools

Tool Description
read_file Read file content (returns draft if one exists)
get_file_info Get file metadata (size, author, dates, permissions)
get_file_history Get Git commit history for a file
get_file_diff Get diff between a commit and HEAD
get_file_at_commit Get file content at a specific commit

File Write Tools

Tool Description
create_file Create a new file
update_file Update an existing file
delete_file Delete a file
move_file Move a file to a different folder
rename_file Rename a file
upload_file Upload a binary file (base64-encoded)

Folder Tools

Tool Description
create_folder Create a new folder
rename_folder Rename a folder
delete_folder Delete a folder and its contents
move_folder Move a folder to a different location

Draft Tools

Tool Description
save_draft Save or update a draft
publish_draft Publish a draft (three-way merge with HEAD)
delete_draft Discard a draft

Search Tools

Tool Description
search Search documents (fulltext, semantic, or hybrid)
search_status Get search index status for all repositories

Permissions

MCP tools enforce the same permission model as the REST API. The authenticated user's access level determines which repositories and files they can read or modify. Operations on files the user cannot access return errors.

Session Tracking

Active MCP sessions are viewable at GET /me/mcp-clients, showing:

  • Session ID
  • User email and name
  • Connection timestamp

Sessions are automatically cleaned up when the SSE connection drops.

Example: Asking Claude About Your Wiki

Once connected, you can ask Claude to interact with your wiki naturally:

  • "List all repositories" — calls list_repos
  • "Read the getting-started guide" — calls read_file
  • "Search for deployment instructions" — calls search
  • "Create a new page about CI/CD" — calls create_file
  • "Show me what changed in this file" — calls get_file_history and get_file_diff
