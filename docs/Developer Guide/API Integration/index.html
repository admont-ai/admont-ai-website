API Integration

Admont-AI exposes a REST API that external applications can use to read and write documents, search content, and interact with the AI assistant. The API is documented with OpenAPI 3.1.

API Documentation

The full OpenAPI specification is available at:

  • JSON: GET /swagger/openapi.json
  • Interactive docs: GET /swagger/index.html

Authentication

All authenticated requests require a JWT token in the Authorization header:

Authorization: Bearer <token>

Obtaining a Token

  1. Initiate OAuth flow: GET /auth/login?provider=google&redirect_uri=https://your-app.com/callback
  2. The user is redirected to the OAuth provider.
  3. After authentication, the user is redirected to your redirect_uri with a token query parameter.
  4. Use this token for subsequent API calls.

Available Auth Providers

GET /auth/providers

Returns the list of configured OAuth providers.

Token Lifetime

JWTs expire after 24 hours. When a request returns 401 Unauthorized, the token has expired and the user must re-authenticate.

Core Operations

List Repositories

GET /repos

Returns all repositories the user can access.

Read a File

GET /repos/:repo/file/*path

Returns the file content. If the user has a draft, the draft content is returned instead.

Create a File

POST /repos/:repo/file/*path
Content-Type: application/json

{ "content": "# New Document\n\nContent here." }

Update a File

PUT /repos/:repo/file/*path
Content-Type: application/json

{ "content": "Updated content." }

Delete a File

DELETE /repos/:repo/file/*path

Search Documents

POST /repos/search
Content-Type: application/json

{
  "repos": [{ "name": "my-docs", "path": "" }],
  "query": "search terms",
  "mode": "hybrid",
  "top_k": 10
}

Modes: hybrid (default), fulltext, semantic.

Ask the AI

POST /repos/rag
Content-Type: application/json

{
  "repos": [{ "name": "my-docs", "path": "" }],
  "query": "What is the deployment process?",
  "model": "claude-sonnet-4-20250514",
  "top_k": 10
}

Returns an answer with sources citing which documents were used.

File Operations Reference

Method Path Description
GET /repos/:repo/file/*path Read file
POST /repos/:repo/file/*path Create file
PUT /repos/:repo/file/*path Update file
DELETE /repos/:repo/file/*path Delete file
PATCH /repos/:repo/file/*path Move file
PUT /repos/:repo/rename/*path Rename file
POST /repos/:repo/upload/*path Upload binary file
GET /repos/:repo/fileinfo/*path File metadata
GET /repos/:repo/filehistory/*path Commit history
GET /repos/:repo/filediff/*path Diff vs HEAD
GET /repos/:repo/fileatcommit/*path Content at commit

Draft Operations

Method Path Description
PUT /repos/:repo/draft/*path Save draft
POST /repos/:repo/draft/publish/*path Publish (three-way merge)
DELETE /repos/:repo/draft/*path Discard draft

Error Handling

  • 401 Unauthorized — Token missing, invalid, or expired.
  • 404 Not Found — File doesn't exist, or you don't have permission (404 is returned instead of 403 to prevent path enumeration).
  • 400 Bad Request — Invalid request body.
  • 500 Internal Server Error — Server error (check logs).

All error responses include a JSON body with error and optionally detail fields.
